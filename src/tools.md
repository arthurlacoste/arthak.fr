# Tools

These are the apps I use day to day for my projects, websites, scripts, and experiments. It is a mix of dev tools, design tools, productivity tools, and analytics.

## Applications (macOS)

- **[Ollama](https://ollama.com/)**: For testing local models, mostly around code, agents, and workflows without depending only on external APIs.
- **[Dropbox](https://www.dropbox.com/)**: Storage and synchronization for projects, notes, exports, website files, and working folders.
- **[FileZilla](https://filezilla-project.org/)**: Occasional FTP/SFTP: 0.40 h measured. Not the sexy tool, but sometimes the shortest path to move a file.
- **[OmniDiskSweeper](https://www.omnigroup.com/more)**: Disk cleanup: 0.07 h measured. Useful when the Mac starts turning into a basement full of boxes.
- **[Sequel Ace](https://sequel-ace.com/)**: MySQL/MariaDB client: 0.02 h measured. Quick inspection of databases, tables, content, and occasional debug.
- **[Opencode](https://opencode.ai/)**: For testing models, code generation, and automated workflows.
- **[Visual Studio Code](https://code.visualstudio.com/)**: Main editor over the last 60 days: 12.37 h measured. Code, Markdown, workers, websites, scripts, back offices.
- **[Ghostty](https://ghostty.org/)**: Main terminal over the last 60 days: 7.31 h measured. Git, local commands, agents, scripts, tests, logs.

## CLI and dev tools

- **[git](https://git-scm.com/)**: Version things properly. Go back when needed. Test without breaking everything.
- **[GitHub CLI](https://cli.github.com/)**: Read issues, create tickets, inspect PRs, follow checks, and push changes faster than through the web interface.
- **[npm / Node.js](https://nodejs.org/)**: Frontend scripts, tooling, packages, workers, small local servers, automations.
- **[Cloudflare Workers](https://developers.cloudflare.com/workers/)**: URL rewrites, cache, edge rules, headers, canonical URLs, routing, and small treatments in front of websites.
- **[Tailscale](https://tailscale.com/)**: Private access to machines and services without exposing random things to the Internet.

## Web stack and infrastructure

- **[Cloudflare](https://www.cloudflare.com/)**: DNS, cache, security rules, redirects, subdomain protection, asset optimization.
- **[OVH](https://www.ovhcloud.com/fr/)**: Historical shared hosting, DNS/domain management, and some web bricks.
- **[GitHub](https://github.com/)**: Code, issues, bug tracking, feature ideas, change history.
- **[Google Analytics 4](https://analytics.google.com/analytics/web/)**: Traffic measurement and global website tracking, when quotas and exports do not break everything.
- **[Search Console](https://search.google.com/search-console/about)**: Concrete SEO reading: indexed pages, queries, errors, sitemap, English pages, Google signals.

## Studio Pixel

- **[Google Ads](https://ads.google.com/)**: Acquisition for Studio Pixel, campaign tests, and local performance tracking.
- **[Jotform](https://www.jotform.com/)**: Client forms, project requests, consent, information collection, and attachments.
- **[Instagram](https://www.instagram.com/)**: Living portfolio, contests, flash sheets, stories, announcements, social proof, and direct contact.
- **[Stripe](https://stripe.com/fr)**: Online payments, deposit thinking, payment links, and potential automation.
- **[Alma](https://www.almapay.com/fr-FR)**: Payment in four installments.
- **[PostHog](https://posthog.com/)**: Possible product analytics to track application usage, understand user paths, and inspect errors.

## Podcast and content

- **[Auphonic](https://auphonic.com/)**: Audio post-production, normalization, export, metadata, chapters, and formats.
